  • CSS Grids
    CSS Grids use a grid-based layout system (rows and columns) that allows the user to easily create a fluid design for their web site which is useful for adapting to many different contexts. This layout system uses CSS instead of traditional HTML positioning techniques like floating. It also allows layering of items on the grid thus a two-dimensional layout unlike flexbox which we covered earlier in the term which is only one-dimensional. CSS Grids allows use to size and position using CSS inside our grid with no changes to HTML.
    CSS Grid Terminology:
    display: grid; Creates the grid container. Similar to flexbox, all items in the grid container become grid items. 
    	.gridContainer { display: grid; }
    grid-template-columns Defines the number and size of grid columns. Example #1 below defines 2 colums with 25% and 75% column sizes respectively. Example #2 uses the fr unit. Fr is a fractional unit where 1fr is for 1 part of the available space so the example would split the space into 7 total fractions spaced as specified. 
    	* Example #1 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 25% 75%;}

    	* Example #2 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 1fr 4fr 2fr;}
    Example #2 layout:
    grid-template-rows Defines the number and size of grid rows. Example #1 below uses the keyword auto. This sizes the rows automatically based on the container size. In other words, it bases the number and size of rows on how many are actually created and how much data is in each row. Example #2, uses the repeat notation to set up each column to contain 2 fractions (rows). 
    	* Example #1 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 25% 75%; 
	 grid-template-rows: auto;}

    	* Example #2 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 1fr 4fr 2fr; 
	 grid-template-rows: repeat(3, 2fr);}
    Example #2 layout:
    grid-template-areas Defines a grid item's size and location/placement. It allows you to map out your grid. (This is actually the shorthand property for grid-row-start, grid-column-start, grid-row-end and grid-column-end.). Notice in Example #1 below, we have a grid that is 2x3 with the header and footer taking up all columns and reference and content split into 2 columns. With grid-template-rows being set to auto, all areas will grow based on content entered. 
    	* Example #1 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 25% 75%; 
	 grid-template-rows: auto;
	 grid-template-areas: 
	 	"header header"
		"reference content"
		"footer footer";}
	.header {grid-area: header; 
		background: blue;}
	.reference {grid-area: reference; 
		background: red; }
	.content {grid-area: content; 
		background: green; }
	.footer grid-area: footer; 
		background: yellow; }

    	* Example #2 */
	.gridContainer { 
	 display: grid; 
	 grid-template-columns: 1fr 4fr 2fr; 
	 grid-template-rows: repeat(3, 2fr); 
	 grid-template-areas:
		"word definitions example;}
	.word {grid-area: word;}
	.def {grid-area: definitions;}
	.ex {grid-area: example;}

    grid-gap Creates space between columns and/or rows. If grid-gap is used, it will put the gap (or gutter) between both columns and rows. You can also use grid-column-gap for gaps between columns only or grid-row-gap for gaps between rows only. Gaps can be a length or a percentage, but you cannot use the fr unit.

  • Flexbox
    Flexbox provides an efficient way to layout, align and distribute items in a specified container. Flexbox is more appropriate for smaller-scaled layouts. CSSGrid, which we will be looking at later in the term, is intended for larger-scaled layouts. Flexbox and CSSGrid are alternatives to the traditional HTML positioning techniques.

    Basically, the flexbox premise is a container with items placed within that container in a certain order and a specified alignment. Imagine you want to hang some pictures on a section of a wall. The wall section would become your flex-container and each picture would become a flex-item. The final picture layout on the wall section is determined by how big is the wall section, how many pictures are to be put on the wall section, and how much space do you want each picture to take up. You also need to decide if the pictures are going to be hung horizontally or vertically.
    NOTE: We will be doing more with flexbox when we cover Media Queries later in the term.

    Container Properties
    display:
    flex
    /* or inline-flex */
    flex-direction:
    row | row-reverse | column | column-reverse
    /* vertical or horizontal */
    flex-wrap:
    nowrap | wrap | wrap-reverse
    /* Items will by default try to fit on one line but this can be changed to include wrapping if needed */
    flex-flow:
    flex-direction || flex-wrap
    /* Shorthand for flex-direction and flex-wrap (default is row nowrap) */
    justify-content:
    flex-start | flex-end | center | space-between | space-around | space-evenly
    /* Alignment along the main axis */
    align-items:
    flex-start | flex-end | center | baseline | stretch
    /* Alignment along the cross axis */
    Item Properties
    flex:
    <number>    <number>    <number> | auto
    /* Shorthand for flex-grow, flex-shrink, and flex-basis which can be set separately.
    flex-grow (default is 0): indicates ability to grow; if set to 1 for all flex-items, the space will be distributed equally to all children
    flex-shrink (optional/default is 1): indicates if flex-item can shrink if necessary
    flex-basis (default is auto): defines size of item before extra space is distributed; auto based on current height and width; content based on items content (not well supported) */
    align-self:
    auto | flex-start | flex-end | center | baseline | stretch
    /* Overrides default alignment for an individual item */
    Flexbox Example
    In this simple example, notice that the flex-container (.wall) must have the display property set to flex (or inline-flex). All flex-items must be inside the flex-container. For each flex-item (.picture#), the flex property (flex-grow option only) is set how large this item can grow. For example, based on the number of flex-items and their sizes, this container (.wall) would split the width of the screen into 7 (1 + 4 + 2) sections. .picture1 only takes up 1 section whereas .picture2 takes up 4 sections and .picture3 takes up 2 sections. NOTE: Borders were added for visual clarity.
    <head>
<style>
 .wall { display: flex; }
 .picCommon { padding: 15px; border: thin solid black;}
 .picture1 { flex: 1;}
 .picture2 { flex: 4;}	
 .picture3 { flex: 2;}		
 img { width: 150px; }
</style>
</head>
<body>
 <section class="wall">
  <div class="picture1 picCommon"><img src="../../images/canada.png"></div>
  <div class="picture2 picCommon"><img src="../../images/canada.png"></div>
  <div class="picture3 picCommon"><img src="../../images/canada.png"></div>
 </section>
</body>
    Change 1
    By adding flex-flow: column nowrap; to the container, notice the difference in appearance.
    <style>
 .wall { display: flex; flex-flow: column nowrap;}
 .picCommon { padding: 15px; border: thin solid black;}
 .picture1 { flex: 1;}
 .picture2 { flex: 4;}	
 .picture3 { flex: 2;}		
 img { width: 150px; }
</style>
    Change 2
    Return to horizontal alignment flex-flow: row nowrap;, change width of images to 20% which will make each picture a different size (since the individual space is different), and then add align-self: for each picture item.
    <style>
 .wall { display: flex; flex-flow: row nowrap;}
 .picCommon { padding: 15px; border: thin solid black;}
 .picture1 { flex: 1; align-self: flex-start;}
 .picture2 { flex: 4; align-self: center;}	
 .picture3 { flex: 2; align-self: flex-end;}		
 img { width: 20%; }
</style>
